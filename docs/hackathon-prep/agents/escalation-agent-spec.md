## What it does — the two faces

The same agent process plays two roles:

1. **The Watcher** (Saturday afternoon → Sunday demo) — runs 18+ hours, polls the CRM, narrates events, places calls when needed. This is the **ElevenLabs bounty submission**.
2. **The phone caller** (live demo moment) — when triggered, places the call to a council number, lets ElevenLabs Conv AI speak, logs the conversation.

Both faces share the same process (`src/agents/watcher.py`) and the same session log.

## Trigger condition — when does a call go out?

```python
def should_call(issue) -> bool:
    if issue.submission_status != "submitted":  return False
    if issue.council_response_received:         return False
    days_open = (now - issue.submitted_at).days
    if days_open < SLA_DAYS[issue.category]:    return False
    if issue.id in self.already_called_today:   return False
    if issue.severity_score < 5:                return False
    return True
```

SLA defaults (in days, per category):

| Category | SLA (days) |
|----------|-----------|
| Waste and Fly-Tipping | 5 |
| Roads and Highways (potholes) | 14 |
| Street Lighting | 7 |
| Antisocial Behaviour | 3 |
| Default | 14 |

**For the demo:** we cheat the clock. The orchestrator's `DEMO_TIME_FACTOR=86400` env var makes "14 days" elapse in 14 seconds during the demo. Honest disclosure in the README — production uses real time.

## The spoken script (locked)

When the call connects, the ElevenLabs Conv AI agent (already configured per `setup/elevenlabs-setup.md`) reads a dynamic first message:

```
Hi, this is an automated call on behalf of {resident_first_name},
a resident in {borough}. I'm following up on a FixMyStreet report —
ticket {short_ref} — about {category_human} on {street_name},
filed {days_open} days ago. Could you confirm the current status?
I'll only need a minute of your time.
```

**Variables filled from the ticket.** If a field is missing, fall back to the literal string `"a resident"` or `"the issue"`.

The system prompt for the agent (already in ElevenLabs dashboard) instructs the agent to:
- Be brief, polite, identify as automated, never claim to be human
- If asked who is calling: *"This is the Sorted, an automated assistant calling on behalf of a London resident."*
- If pressed for the resident's contact: *"I don't share resident contact details. I can pass any update back to them."*
- If told the issue is resolved: thank them, log the outcome, end the call.
- If put on hold: wait politely up to 60 seconds, then *"I'll call back later — thank you."*

## Session log — the bounty artifact

Every event the agent observes is one row in `watcher_session_log`:

```sql
CREATE TABLE watcher_session_log (
    id INTEGER PRIMARY KEY,
    session_id TEXT NOT NULL,        -- one session = one run, used for resume
    ts TEXT NOT NULL,                 -- ISO 8601 UTC
    event_type TEXT NOT NULL,         -- new_report | sla_breach | outbound_call_start
                                       -- | outbound_call_outcome | voice_question
                                       -- | watcher_heartbeat | seed_event
    issue_id INTEGER,                 -- nullable
    payload_json TEXT NOT NULL        -- whatever's relevant for replay
);
```

Heartbeats fire every 60 seconds with `event_type='watcher_heartbeat'`. This is how we prove the agent ran continuously across the 1h 11min minimum.

Submission artifact (`watcher_session.json`) is generated by exporting this table to JSON before submission. Format already documented in `bounties/elevenlabs-prize-strategy.md`.

## Cursor build prompt (paste at 15:30)

```
GOAL: Build the Sorted Watcher — a long-running process that:
1. Polls processed_issues every 30 seconds for new rows / status changes
2. Logs every observation to watcher_session_log
3. Detects SLA breaches and triggers outbound phone calls
4. Stays alive across the entire hackathon afternoon + Sunday morning

CONSTRAINTS:
- File: src/agents/watcher.py (entrypoint: python -m src.agents.watcher)
- Uses sqlite3 from the same db file as the rest of the pipeline
- trigger_call function is already in src/integrations/twilio_eleven.py — DO
  NOT reimplement; import and use.
- Heartbeat every 60s. Skipped if no work — but heartbeat row still goes in
  the log.
- SLA values: see escalation-agent-spec.md (this file). DEMO_TIME_FACTOR env
  var multiplies the "days" comparison for demos.
- ON CRASH: re-launch reads the latest session_id from
  watcher_session_log, resumes appending to it. Never start a new session
  unless --new-session flag is passed.
- Logging: every poll cycle = one heartbeat event. Every new processed_issues
  row = one new_report event. Every triggered call = one outbound_call_start
  event with payload {phone, council, ticket_id, dialled_at}, followed by
  an outbound_call_outcome event after the call.
- Export: `python -m src.agents.watcher --export-session` writes
  watcher_session.json in the format from
  bounties/elevenlabs-prize-strategy.md.

ACCEPTANCE:
- Start with: `python -m src.agents.watcher` in the foreground.
- After 2 minutes, watcher_session_log has at least 2 heartbeat rows.
- Insert a row into processed_issues with status='submitted' and submitted_at
  set to 30 days ago — within 30-60s a 'sla_breach' event appears in the log
  with the issue id.
- Stop the process and re-run — it picks up the same session_id and continues.

CONTEXT:
- bounties/elevenlabs-prize-strategy.md is the strategic frame
- setup/twilio-setup.md has trigger_call wiring details
- setup/elevenlabs-setup.md has the Conv AI agent's system prompt
- decisions-locked.md says NO async, vanilla Python only (use a while
  True + time.sleep loop, not asyncio)
- Crashes must not lose context — session_log IS the memory
```

## Failure modes + recovery

| Symptom | Likely cause | Fix |
|---------|--------------|-----|
| Watcher silently dies after 30 min | Unhandled exception in poll loop | Wrap poll in try/except, log to session_log with `event_type='watcher_error'`, sleep 60s, continue |
| Phone call places but agent doesn't speak | ElevenLabs agent ID wrong | Re-check `ELEVENLABS_AGENT_ID` |
| SLA breach loop fires every poll | `already_called_today` not persisted | Persist to SQLite, not in-memory |
| Session log fills disk | Heartbeats accumulating | Rotate weekly — for hackathon, fine |
| Judges' questions get generic answers | Agent context not loaded with session log | Pass last 200 log entries as context with the agent's system prompt on each query |
