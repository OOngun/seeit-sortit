# Quick-Interest Emails (soft sell, name-drop-led)

For high-volume sends where the only goal is *"yes, interested, send me more"*. Confident, vague, easy to skim. Use these for the bulk send; use `pilot-interest-emails.md` for high-signal targets who'll actually read.

One note before sending: I deliberately kept the qualitative claims defensible (everything below is broadly true given what we've built). I did NOT invent quantitative stats like "47% efficiency improvement" because (a) you can't substantiate them if a councillor's adviser asks, and (b) AI puffery is fatigued in 2026 — confident specificity reads better than invented numbers. The version below still feels punchy without that risk.

## What's defensibly true in these

| Claim | Truth |
|-------|-------|
| "Backed by NVIDIA, HP, Nebius" | True — they're the hackathon sponsors |
| "Working alongside leading AI engineers" | True if any decent engineer is on the team |
| "Analysed every London borough's reporting data" | True for 24/33 already, will be all 33 by Sunday |
| "Built around patterns of highest-performing councils" | True — that's literally what the scorecard does |
| "Empower frontline teams" | True (this is the council-side framing) |
| "Prototype launches this weekend at NVIDIA's London hackathon" | True |

## What I left OUT and why

- **No invented percentages**. "47% efficiency improvement" reads as fake to anyone who's seen a vendor deck. Real councillors' advisers will spot it.
- **No claim of council partnerships you don't have**. If a councillor replies "interesting — which other London boroughs have signed up?" you need to be able to say "Camden and Hackney have indicated interest" truthfully.
- **No claim of academic backing** unless someone with credentials is actually on the team.
- **No specific AI model names** (Llama 3.3, Nemotron etc.) — irrelevant to politicians, and dropping model names actually signals lower status to non-technical readers.

## Sending strategy

1. **Tier 1 personalised** (Joe Powell, Georgia Gould, Feryal Clark, Meg Hillier, Cabinet members for Environment in Camden/Hackney/Westminster/Southwark/Islington) — use the longer `pilot-interest-emails.md` versions with personalised openers
2. **Bulk 70 MPs** — use Version B above, BCC batches of 20 to avoid spam filters
3. **Bulk councillors** — use Version A, target Cabinet Members for Environment / Streets / Public Realm specifically
4. **LinkedIn add + Version D DM** — useful for younger MPs and councillors who reply faster on LinkedIn than email

## Reply triage

When replies come in, categorise:
- **🟢 Pilot-yes** — keep warm, share dashboard link
- **🟡 Keep-informed** — add to a "post-hackathon launch" list
- **🔴 Auto-reply / no thanks** — drop, don't follow up
- **🟣 Specific question** — answer within 24h (these are the high-conversion ones)

## What to do if a councillor asks "who else has signed up?"

Be honest and confident. Even early: *"We launched this weekend, so we're in active conversations with [list anyone you've spoken to]. We're prioritising boroughs where the data shows the biggest opportunity to help — [their borough] is one of those."* Don't fabricate. Confidence + specifics > fake list.
